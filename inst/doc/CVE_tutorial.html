Cancer Variant Explorer (CVE) tutorial

Andreas Mock
Cancer Research UK Cambridge Institute, University of Cambridge

2016-06-27

1) Introduction

The goal of the tutorial is to present the functionality of the CVE package. CVE can be installed via Bioconductor

source('http://www.bioconductor.org/biocLite.R')
biocLite('CVE')

and once installed loaded into the R session by

library(CVE)

2) Oncotator Annotation

CVE leverages output files of the Oncotator Annotation Tool1 to prioritise variants. The CVE package includes the example Oncotator output file oncotator_example for the 1087 mutations prioritised in the melanoma case study. CVE can either work with a data frame containing the output file of one sample or a list containing multiple files.

3) Reading data and opening the shiny app

The function openCVE opens the CVE shiny app. It requires the oncotator output file as a data frame (output of 1 sample) or a list (output of multiple samples) and the corresponding sample name(s).

openCVE(oncotator_example, sample_names='case study')

To showcase the modularity of CVE, we added an extension for WGCNA

openCVE(oncotator_example, sample_names='case study', extension='WGCNAmelanoma')

4) Priorising variants with CVE

For details regarding prioritising variants, please refer to the paper 2.
